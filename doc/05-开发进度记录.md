### 2.5 CathSim NavigationEngine 桥接

新增后端模块：

| 文件 | 作用 |
|------|------|
| `services/navigation_engine.py` | 封装 CathSim MuJoCo 环境，提供 `reset()`/`step()` |
| `services/session_manager.py` | 多会话管理，支持并发导航实例 |

`NavigationEngine` 核心功能：

- 封装 `cathsim.dm.make_dm_env` 创建环境
- `reset()` 返回 `NavigationState` 初始状态
- `step(delta_push, delta_rotate)` 执行物理仿真步
- `get_safety_status(state)` 返回安全级别
- 提取标准化状态字段：
  - `tip_position` - 导丝头端位置
  - `tip_direction` - 导丝头端方向向量
  - `velocity` - 头端速度
  - `contact_force` - 接触力
  - `episode_length` - 当前 episode 步数
  - `target_position` - 目标位置
  - `reward` / `done` - 强化学习信号

`SessionManager` 核心功能：

- 线程安全的会话创建/销毁
- 自动过期清理（默认 1 小时）
- 最大并发会话限制（默认 10）
- 会话元数据跟踪（创建时间、步数、episode 数）

### 2.10 B-spline 路径平滑

更新 `services/path_planner.py`：

- `smooth_path()` 方法：基于 scipy.interpolate.splprep 的 B-spline 平滑
- `_compute_max_curvature()` 方法：计算路径最大曲率
- `PathResult` 增加字段：`smooth_waypoints`, `smooth_length_mm`, `max_curvature`
- `plan()` 方法支持 `smooth` 参数

平滑算法特性：

- 基于累积弧长参数化
- 支持可调平滑因子 `smoothing_factor`（0.0 = 插值，>0 = 平滑）
- 自动计算输出点数（默认 2x 输入）
- 计算路径曲率用于安全评估

API 更新：

| 方法 | 路径 | 新增参数 |
|------|------|----------|
| `POST` | `/api/v1/path/plan` | `smooth: bool` |

响应新增字段：

- `smooth_waypoints` - 平滑后的路径点
- `smooth_length_mm` - 平滑路径长度
- `max_curvature` - 最大曲率 (mm⁻¹)

### 7.2 修复内容

本阶段发现并修复了两个集成问题：

1. 高精度 `visual.stl` 不应注册进 MuJoCo XML

   原因：MuJoCo STL decoder 对单个 STL 面数有限制，高精度血管表面更适合保留给外部渲染器，不适合直接作为 MuJoCo mesh asset 编译。

   处理：

   - `tools/vhacd_decompose.py` 保留 `visual.stl` 文件导出
   - MuJoCo XML 只注册 `hull_*.stl` 碰撞凸包
   - 当前 `case_001_vpp.xml` 已移除 visual mesh/geom 引用

2. `Phantom` 需要兼容无 visual mesh 的 VPP XML

   处理：

   - `src/cathsim/dm/components/phantom.py` 的 `set_rgba()`、`set_scale()` 在找不到 visual mesh/geom 时跳过
   - `case_001_vpp.xml` 对应 `meshes/case_001/` 的目录命名已兼容

3. Python 依赖版本需要固定

   处理：

   - `pyproject.toml` 固定 `dm-control==1.0.14`
   - `pyproject.toml` 固定 `mujoco==2.3.7`
   - 补充 `pympler`、`pyvista`、`vhacdx`、`scipy`

   背景：`dm_control 1.0.41` 与 `mujoco 3.9.0` 在当前环境中触发 `flex_bandwidth` 索引不兼容；项目 `environment.yml` 原本使用的是 `dm-control==1.0.14` 与 `mujoco==2.3.7`。
