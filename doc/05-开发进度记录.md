### 2.5 CathSim NavigationEngine 桥接

新增后端模块：

| 文件 | 作用 |
|------|------|
| `services/navigation_engine.py` | 封装 CathSim MuJoCo 环境，提供 `reset()`/`step()` |
| `services/session_manager.py` | 多会话管理，支持并发导航实例 |

`NavigationEngine` 核心功能：

- 封装 `cathsim.dm.make_dm_env` 创建环境
- `reset()` 返回 `NavigationState` 初始状态
- `step(delta_push, delta_rotate)` 执行物理仿真步
- `get_safety_status(state)` 返回安全级别
- 提取标准化状态字段：
  - `tip_position` - 导丝头端位置
  - `tip_direction` - 导丝头端方向向量
  - `velocity` - 头端速度
  - `contact_force` - 接触力
  - `episode_length` - 当前 episode 步数
  - `target_position` - 目标位置
  - `reward` / `done` - 强化学习信号

`SessionManager` 核心功能：

- 线程安全的会话创建/销毁
- 自动过期清理（默认 1 小时）
- 最大并发会话限制（默认 10）
- 会话元数据跟踪（创建时间、步数、episode 数）

### 2.10 B-spline 路径平滑

更新 `services/path_planner.py`：

- `smooth_path()` 方法：基于 scipy.interpolate.splprep 的 B-spline 平滑
- `_compute_max_curvature()` 方法：计算路径最大曲率
- `PathResult` 增加字段：`smooth_waypoints`, `smooth_length_mm`, `max_curvature`
- `plan()` 方法支持 `smooth` 参数

平滑算法特性：

- 基于累积弧长参数化
- 支持可调平滑因子 `smoothing_factor`（0.0 = 插值，>0 = 平滑）
- 自动计算输出点数（默认 2x 输入）
- 计算路径曲率用于安全评估

API 更新：

| 方法 | 路径 | 新增参数 |
|------|------|----------|
| `POST` | `/api/v1/path/plan` | `smooth: bool` |

响应新增字段：

- `smooth_waypoints` - 平滑后的路径点
- `smooth_length_mm` - 平滑路径长度
- `max_curvature` - 最大曲率 (mm⁻¹)

### 9.2 任务分解

#### 任务 A：`NavigationEngine` 注入 `PathPlanner`

**目标**：让 `NavigationEngine` 在 `reset()` 时接受一条规划路径，并在每步 `step()` 后实时计算路径进度和偏离量。

```python
# 新增接口
engine = NavigationEngine(
    phantom=”case_001_vpp”,
    target=”endpoints_1”,
    assets_dir=”...”,
    planned_path=smooth_waypoints,   # List[List[float]]，单位 m
)
state = engine.reset()
# state.path_progress  → [0.0, 1.0]
# state.path_deviation → 距规划路径最近距离 (m)
```

实现要点：
- `NavigationEngine.__init__` 增加 `planned_path: list[list[float]] | None` 参数
- 构建 `scipy.spatial.KDTree` 用于最近点查询
- `_extract_state()` 中计算 `path_progress`（沿路径弧长累积比例）和 `path_deviation`

#### 任务 B：`NavigationEngine` 计算 `wall_distance` 与 `curvature`

**目标**：每步返回导丝尖端的壁面距离和局部曲率。

```python
# NavigationState 新增字段
wall_distance: float = 0.0   # 单位 m（由 MuJoCo 接触几何估算）
curvature: float = 0.0       # 单位 m⁻¹（5 点差分法计算尖端曲率）
```

实现要点：
- `wall_distance`：从 MuJoCo `physics.data.contact` 中读取最近接触点距离；若无接触则设为大值（如 0.05m）
- `curvature`：维护尖端位置历史队列（5 帧），用 3 阶多项式或有限差分估算曲率

#### 任务 C：`NavigationState` 增加 `safety_status` 与 `tip_quaternion`

**目标**：将安全状态纳入 `NavigationState`，由 `NavigationEngine` 内部在 `_extract_state()` 中计算，而非由外部调用者手动调用 `get_safety_status()`。

```python
@dataclass
class NavigationState:
    ...
    wall_distance: float = 0.0
    curvature: float = 0.0
    path_progress: float = 0.0
    path_deviation: float = 0.0
    safety_status: str = “STANDBY”
```

| wall_distance < 0.0005m | `COLLISION_STOP` |

### 10.3 关键实现说明

1. **wall_distance（壁面距离代理）**
   - 从 `physics.data.contact.dist` 读取接触间隙，惩罚穿透（clip 到 0），无接触时返回哨兵值 `MAX_WALL_DISTANCE = 0.05m`

2. **curvature（曲率）**
   - 基于最近 3 个尖端位置的 Menger 曲率（外接圆半径倒数），单位 m⁻¹

   - deviation = 到最近顶点欧氏距离（米）

   - 从尖端 body 的 `physics.data.xquat` 读取，MuJoCo `[w,x,y,z]` 重排为协议约定的 `[x,y,z,w]`
   - 读取失败回退单位四元数

### 11.3 关键设计说明

1. **state_batch 触发方式**：采用会话级 `batch_mode` 开关（`session_start` 时指定），而非新增客户端消息类型。`batch_mode=false`（默认）时仍发 `state_update`，旧测试零回归；Godot 渲染导丝 Tube 时用 `batch_mode=true`。

2. **导丝 body 提取**：遍历 `model.ngeom`，按 `model.id2name(geom_bodyid, "body")` 含 `"guidewire"` 过滤，返回 `{pos, quat}`（四元数 `[w,x,y,z]`→`[x,y,z,w]`），自基座到尖端有序，供 Tube 渲染。

3. **path_request 无需会话**：客户端可在创建导航会话前先请求路径；`PathPlanner` 按 `case_id` 用 `lru_cache` 复用，坐标为 LPS 毫米。

4. **risk_score 与 safety_status 并存**：`safety_status` 仍由壁面距离状态机给出（安全状态灯），`risk_score`（0–1，四指标加权）用于 HUD 风险计。二者解耦，符合 doc §1.4 `safety.status` 与 `safety.risk_score` 分列的设计。
