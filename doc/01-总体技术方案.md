# 血管介入仿真数字孪生平台 — 总体技术方案

> 版本：v1.0 | 日期：2026-06-16  
> 基于 CathSim (MuJoCo) + Vascular Path Planner Sim 融合架构

---

## 一、项目目标

构建一套面向血管介入导航场景的高保真数字孪生仿真平台，实现**术前规划 → 术中导航 → 技能评估**全流程覆盖。

### 设计原则

| 原则 | 说明 |
|------|------|
| **物理保真** | 基于 MuJoCo 物理引擎，确保导丝动力学和碰撞的真实性 |
| **渲染保真** | Godot 游戏引擎提供高帧率 GPU 渲染与 X-ray 着色器效果 |
| **模块解耦** | 微服务架构，各模块独立部署、独立演进 |
| **Sim2Real** | 机器人模型（CathBot）可迁移至真实硬件 |

### 系统定位

```
Godot 渲染交互层    →  视觉验证 & 客户演示
CathSim 物理引擎层  →  导丝物理 & RL 训练
VPP 数据预处理层    →  中心线提取 & 路径规划
FastAPI 微服务层    →  数据桥接 & 状态管理
```

## 三、核心模块设计

### 3.1 物理引擎层（CathSim）

以 CathSim 的 MuJoCo 环境为核心，提供：

```
┌──────────────────────────────────────────────────┐
│              CathSim 环境 (dm_env)                 │
│                                                    │
│  Phantom(low_tort.xml)                             │
│  ├─ 235 个凸包 STL → MuJoCo geoms                  │
│  └─ 血流阻力计算                                   │
│                                                    │
│  Guidewire(n_bodies=80)                           │
│  ├─ 80 节关节链 (hinge/torsion)                    │
│  ├─ 每节刚度(k=0.5) / 阻尼(b=0.2)                  │
│  └─ 与血管壁凸包碰撞                               │
│                                                    │
│  Tip(n_bodies=4)                                   │
│  └─ 尖端 4 节精密关节                              │
│                                                    │
│  CathBot (摩擦驱动)                                │
│  ├─ 驱动轮与导丝接触                               │
│  └─ 力反馈 (推向/旋转)                             │
└──────────────────────────────────────────────────┘
```

#### 状态空间（返回给 Godot）

```python
{
    "wire_pos": [x, y, z],            # 导丝尖端位置
    "wire_dir": [dx, dy, dz],         # 导丝尖端方向单位向量
    "joint_pos": [θ₁, θ₂, ..., θ₈₀],  # 80 节关节角度
    "contact_force": 0.0,             # 当前接触力 (N)
    "wall_distance": 2.5,             # 壁面最小距离 (mm)
    "velocity": 0.3,                  # 推进速度 (mm/s)
    "progress": 0.45,                 # 路径完成度 (0~1)
    "safety_status": "SAFE_NAV"       # 安全状态
}
```

#### 动作空间（从 Godot 接收）

```python
{
    "delta_push": 0.5,    # 推进增量 (mm), 范围 [-1.0, 1.0]
    "delta_rotate": 0.1   # 旋转增量 (rad), 范围 [-0.5, 0.5]
}
```

### 3.2 路径规划服务（VPP 复用）

复用于 `vascular_path_planner_sim/Vascular-Path-Planning/` 的工业级规划管线：

```
CTA 影像
   │
   ▼ 3D Slicer 分割 + VMTK 中心线
   ▼
147 条 mrk.json 曲线
   │
   ▼ merge_centerline_curves.py
   ▼
Centerline_curves_merged.vtk (10,705 点)
   │
   ▼ centerline_graph.py (Union-Find + KDTree)
   ▼
centerline_vessel_net.json (10,561 节点, 21,120 边)
   │
   ▼ A* 规划器 (曲率权重 + 壁面惩罚)
   ▼
原始路径 → BSplineSmoother (splprep 约束模式)
   ▼
平滑路径 (最大曲率 < 0.1 mm⁻¹)
```

### 3.3 风险评估体系

| 指标 | 计算方法 | 权重 | 阈值 | 安全等级 |
|------|---------|------|------|---------|
| 壁面距离 | MuJoCo 接触几何 → 最近点 | 0.4 | < 0.5mm → CRITICAL | 四级分级 |
| 曲率 | 5 点差分法 | 0.3 | > 0.15 → CRITICAL | STANDARD (绿色) |
| 速度 | | 0.2 | > 8mm/s → WARNING | SAFE_NAV (绿色) |
| 路径偏离 | 距规划路径最近距离 | 0.1 | > 3mm → CRITICAL | DANGER_WARN (黄色) |
| | | | | COLLISION_STOP (红色) |

## 五、安全与约束

### 安全分级状态机

```
                ┌──────────┐
                │ STANDBY  │ ← 启动/复位
                └────┬─────┘
                     │ 有控制输入
                     ▼
    ┌──────────────────────────────────┐
    │           SAFE_NAV               │ ◄── 正常导航（绿色）
    │ 壁距 >= 1.0mm, 曲率 < 0.10      │
    └────────┬─────────────────────────┘
             │ 1.0mm > 壁距 >= 0.5mm │ 壁距恢复
             ▼                       │
    ┌──────────────────────────────────┐
    │         DANGER_WARNING           │ ◄── 接近壁面（黄色）
    │ 限速 50%, 闪烁 HUD 警示          │
    └────────┬─────────────────────────┘
             │ 壁距 < 0.5mm         │ 壁距恢复
             ▼                       │
    ┌──────────────────────────────────┐
    │        COLLISION_STOP            │ ◄── 触壁制动（红色）
    │ 立即制动, 禁止推进, 仅可后退      │
    └──────────────────────────────────┘
```

### 运动学约束

| 参数 | 值 | 说明 |
|------|-----|------|
| 最大推进速度 | 5 mm/s | 防止穿透 |
| 最大旋转速度 | 0.5 rad/s | 防止跳跃 |
| 单帧最大位移 | 0.5 mm | 防穿透保护 |
| 最小转弯半径 | 5 mm | 导丝弯曲限制 |
