## 二、血管模型数据规格

### 2.1 CathSim 凸包模型

```
格式: STL (Binary)
来源: 硅胶体模 3D 扫描 → 网格 → V-HACD 凸包分解
坐标: MuJoCo 坐标系
数量: 235 个凸包 (low_tort)
特点:
  - 每个凸包 ≤ 64 顶点 (V-HACD 默认)
  - 凸包近似 → 碰撞检测高效
  - 但丢失了血管曲面细节 (阶梯状)
```

**限制**：

- 凸包间存在缝隙，视觉不连续
- 无法精确计算壁面距离（凸包内壁被近似为平面）
- 无中心线/拓扑信息
- 仅 4 种形态可选

### 2.2 VPP 真实血管模型

```
格式: VTK Polydata (Binary)
来源: 真实患者 CTA → 3D Slicer 分割重建
坐标: LPS (医学标准，与 DICOM 一致)
精度: 312,576 点 / 625,124 三角面
血管树: 完整主动脉弓 + 分支 (头臂干、左颈总、左锁骨下)
```

**优势**：

- 高精度曲面，光滑连续
- 医学标准 LPS 坐标系，可直接与 DICOM 配准
- 含完整血管树分支
- 可与 Slicer 标记点直接对应

### 2.3 转换需求

```
VPP VTK 血管 → CathSim MuJoCo 的转换管线：

blood_vessels.vtk
       │
       ▼ trimesh.load()
VTK → 三角网格
       │
       ▼ V-HACD (v-hacd.exe / pypi vhacdx)
凸包分解 (maxHulls=300, maxVerticesPerHull=64)
       │
       ▼ muojoco_utils / 手写 XML
MuJoCo XML 格式
       │
       ▼ 嵌入 phantom 定义
low_tort_vpp.xml
```

## 三、中心线数据规格

### 3.1 VPP 中心线（来自 Slicer）

```
来源:    3D Slicer "Centerline Extraction" 模块
格式:    .mrk.json (Slicer 标记曲线) + .vtk (合并后)
点数量:  10,705 个 (145 条曲线合并去重后)
曲线数:  145 条 (覆盖血管树所有分支)
流距:    0~423.2 mm (沿血管中心线的弧长)
曲率:    最大 0.527 mm⁻¹
坐标:    LPS (与血管模型一致)
```

### 3.2 图结构

```
格式:    JSON 坐标键值邻接表
节点数:  10,561
边数:    21,120
连通性:  全连通 (所有节点从根可达)
坐标格式: "-1.489951,-268.458984,290.415405" → 字符串键
边权重:  欧几里得距离
```

## 四、导丝模型规格

### 4.1 CathSim 导丝参数

```
类型:     MuJoCo 关节链 (WeldJoint + HingeJoint + TorsionJoint)
节数:     80 (可配置, Guidewire(n_bodies=80))
每节长度: ~5 mm (80 节 → 约 400 mm 总长)
刚度:     k = 0.5 (弯曲刚度)
阻尼:     b = 0.2
质量:     每节 0.001 kg
半径:     0.5 mm (导丝截面)
尖端:     Tip(n_bodies=4), 更精密, 更高刚度
颜色:     RGB (0.9, 0.9, 0.9) 银白色
```

### 4.2 导丝状态观测

```python
# dm_env 观测空间
obs = {
    "joint_pos": np.ndarray(shape=(84,)),  # 80 节 + 4 节尖端 = 84 关节
    "joint_vel": np.ndarray(shape=(84,)),  # 关节速度
}

# 扩展观测 (用于 Godot 渲染)

    "tip_position": np.ndarray(shape=(3,)),     # 尖端世界坐标

    "body_positions": np.ndarray(shape=(84, 3)), # 每节位置 (渲染导丝 Tube)
    "contact_forces": np.ndarray(shape=(84,)),   # 每节接触力
}
```

## 五、动作空间规格

### 5.1 CathSim 原生动作

```python
# dm_env 动作空间
action_spec = {
    "minimum": np.array([-1.0, -1.0]),   # [推拉力, 旋转力]
    "maximum": np.array([1.0, 1.0]),
}

# 动作映射 (CathBot 摩擦驱动)
# action[0] = 推进力系数 → 驱动轮转速
# action[1] = 旋转力系数 → 旋转夹头角度
```

### 5.2 Godot 输入映射

| Godot 输入 | 映射动作 | 范围 | CathSim 动作映射 |
|-----------|---------|------|----------------|
| 左摇杆 Y 轴 | `delta_push` | [-1.0, 1.0] | `action[0] = delta_push` |
| 右摇杆 X 轴 | `delta_rotate` | [-1.0, 1.0] | `action[1] = delta_rotate * 0.5` |
| 左扳机 (LT) | 加速推进 | [0, 1.0] | 速度倍增系数 ×1~×3 |
| 右扳机 (RT) | 精确控制 | [0, 1.0] | 速度衰减系数 ×0.5~×1 |
| A 键 | 急停 | — | `action = [0, 0]` |
| B 键 | 复位 | — | `env.reset()` |

## 七、坐标系统

| 系统 | 坐标约定 | 使用场景 |
|------|---------|---------|
| **LPS** | +X = 左, +Y = 后, +Z = 上 | VPP 所有资产 (3D Slicer 标准) |
| **MuJoCo** | +X = 右, +Y = 前, +Z = 上 | CathSim 物理引擎 |
| **Godot** | +X = 右, +Y = 上, +Z = 前 | 渲染客户端 |

**转换矩阵 (LPS → MuJoCo)**：

```
M = [[-1,  0,  0],
     [ 0, -1,  0],
     [ 0,  0,  1]]
```

**转换矩阵 (LPS → Godot)**：

```
M = [[-1,  0,  0],
     [ 0,  0,  1],
     [ 0,  1,  0]]
```
